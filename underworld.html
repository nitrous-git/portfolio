Game overview

Fiction first, grounded by production

Underworld is a 2D pixel-art action-adventure platformer set in a haunting sci-fi realm deep beneath the surface. You play as a nano-enhanced swordsman, awakened by a mysterious AI after being left for dead in the ruins of a forgotten world. From there, the game pushes a clear fantasy: descend deeper, survive hostile spaces, and uncover what happened to this corrupted underworld.

The tone stays close to the public game pitch: nonlinear exploration, dangerous creatures, hidden paths, upgrades, and a more deliberate combat rhythm inspired by Metroidvania structure and Souls-like pressure. The result feels moody and stylized rather than retro for its own sake.

Production and systems

Technical side of the game

The important technical value of Underworld is that it brings multiple gameplay and presentation layers together into a coherent public build. This is not just a visual mockup. It is a real game page backed by a playable release, which means interaction, pacing, UI, combat clarity, and general usability had to hold up under actual players.

  • Unity + C# development: gameplay code and feature integration in a practical shipped-game context.
  • 2D platformer structure: player movement, traversal flow, enemy encounters, and map progression working as one loop.
  • Comic-book story framing: stylized narrative presentation used to establish tone and identity early.
  • Combat FSM boss logic: encounter behavior built around readable state-driven combat and multi-phase pressure.
  • Store and menu layer: player-facing interface work, navigation, and supporting UX needed for a complete build.
  • Camera work: moment-to-moment readability depends heavily on follow behavior, framing, and platforming clarity.
